# 部署说明：数据全在数据库

本项目已改为**所有业务数据与上传文件均存数据库**，不依赖本地磁盘，适合上传到云平台/学校服务器部署。

---

## 一、数据存储方式

| 类型 | 存储位置 | 说明 |
|------|----------|------|
| 用户、工单、评价等 | **数据库** | MySQL / SQLite 表 |
| 上传图片（头像、工单现场照） | **数据库** | `repairs.StoredFile` 表，不再使用 `media/` 目录 |
| 会话 | **数据库** | Django 默认 `django_session` 表（或 Redis，若后续配置） |

- **不再使用**：项目下的 `media/` 目录、本地文件形式的用户/工单数据。
- **前端**：登录态仅依赖后端 Session（Cookie），不把用户信息写入 localStorage，便于多端/多机一致。

---

## 二、部署到平台时建议

### 1. 使用 MySQL

- 在服务器或平台提供的 MySQL 中创建数据库 **DormFix**（见 `scripts/create_mysql_db.sql`）。
- 在部署环境配置环境变量 **`DB_PASSWORD`**（MySQL root 或业务账号密码）。
- 项目逻辑：若配置了 `DB_PASSWORD`，则使用 MySQL；未配置时本地开发可退化为 SQLite。

### 2. 执行迁移与初始化

```bash
python manage.py migrate
python manage.py init_data   # 可选：初始化测试账号与故障类型
```

### 3. 静态与媒体

- 静态文件：按平台要求用 `python manage.py collectstatic` 收集到指定目录并由 Nginx/平台托管。
- 媒体文件：已改为数据库存储，**无需**再挂载或备份 `media/` 目录。

### 4. 前端

- Vue 前端：`npm run build` 后把 `dist/` 部署到静态托管或同域名下。
- 确保前端请求的 API 地址与后端一致，并带上 Cookie（`credentials: 'include'`）以便 Session 生效。

---

## 三、相关代码说明

- **文件存库**：`repairs/storage_db.py`（`DatabaseStorage`）、`repairs/models.StoredFile`；访问路径形如 `/media/db/<id>/`，由视图从数据库读出并返回。
- **数据库配置**：`dormfix_backend/settings.py` 中 `DATABASES`、`STORAGES`（默认文件存储为 `DatabaseStorage`）。
- **前端不存用户到本地**：登录/个人中心等仅依赖 Session，不写用户信息到 localStorage。

按上述方式部署后，**所有业务数据与上传文件都在数据库中**，无本地文件依赖。
